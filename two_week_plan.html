Week 1 — Build the core package
Day 1
Project setup & solution structure
Create solution with 3 projects: ShadowService (library), ShadowService.Tests, ShadowService.Demo (ASP.NET app). Install xUnit, Moq. Understand .csproj and solution files.
Day 2
Core data model — CapturedRequest
Design the C# record/class: Id, Timestamp, Method, Path, Headers, RequestBody, ResponseBody, StatusCode, DurationMs. Learn records vs classes in C#.
Day 3
IShadowSink interface + in-memory implementation
Define IShadowSink with WriteAsync(CapturedRequest). Build InMemorySink. This is the pluggable storage pattern — learn interfaces and dependency injection.
Day 4
ASP.NET middleware — ShadowMiddleware
Implement IMiddleware. Clone the HttpRequest body stream (streams are one-read-only — this is the key challenge). Capture request, call next(), capture response. No await blocking the main thread.
Day 5
Async background queue
Use Channel<CapturedRequest> or a ConcurrentQueue. Middleware enqueues; a BackgroundService dequeues and calls IShadowSink.WriteAsync. Keeps the request pipeline fast — learn async/await patterns.
Day 6
DI registration + AddShadowService() extension
Create IServiceCollection extension so users just call builder.Services.AddShadowService(). Learn the options pattern (ShadowServiceOptions). Make it feel like a real NuGet package.
Day 7
Unit tests + wire up demo app
Test InMemorySink, the queue, and data model. Wire the middleware into the Demo app. Make a real HTTP call and confirm it gets captured. First big milestone.
✓ Milestone: working core package
Week 2 — Storage sinks, REST API, and NuGet publish
Day 8
File sink (JSON lines)
Implement FileSink: write each CapturedRequest as a JSON line to a .jsonl file. Learn System.Text.Json serialization. Handle file locking safely with SemaphoreSlim.
Day 9
SQL sink with EF Core
Create SqlSink using Entity Framework Core + SQLite. Design the captures table. Learn DbContext, migrations, and async writes. Wire to demo app — swap sinks with one line of config.
Day 10
Filtering & sampling options
Add ShadowServiceOptions: ExcludePaths (e.g. /health), SamplingRate (0.0–1.0), MaxBodySize. Apply filters inside the middleware before enqueuing. Learn Options pattern properly.
Day 11
REST API read-back endpoints
Add a CapturesController to the demo/host app: GET /captures (paged), GET /captures/{id}, GET /captures/stats (count, avg duration, top paths). Learn minimal API vs controller routing.
Day 12
Integration tests + edge cases
Test large bodies, binary content, multipart forms, high concurrency. Use WebApplicationFactory to spin up a real test host. Fix any race conditions in the queue.
Day 13
Package & publish to NuGet
Configure .csproj PackageId, Version, Description, authors. dotnet pack → .nupkg. Publish to nuget.org (free account) or a local feed. Write a README with a code sample.
Day 14
Polish, docs, and demo
XML doc comments on public APIs. GitHub repo with README, architecture diagram, and quick-start. Run the demo app end-to-end and confirm captures appear in all three sinks.
✓ Milestone: published NuGet package
